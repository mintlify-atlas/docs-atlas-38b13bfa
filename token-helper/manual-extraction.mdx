title: "Manual extraction"
description: "Copy your Discord token from browser DevTools and enter it in Deleo manually"
---

<Warning>
  Your Discord token provides full access to your Discord account. Never share it with anyone. Do not paste it into websites, scripts, or applications you do not trust.
</Warning>

If you cannot or do not want to run the Python token-helper script, you can copy your token from browser DevTools and enter it in Deleo manually.

## Option 1: Browser DevTools (recommended for manual extraction)

<Steps>
  <Step title="Open Discord in your browser">
    Go to [discord.com/app](https://discord.com/app) in Chrome, Edge, Brave, or Firefox and log in if prompted.
  </Step>
  <Step title="Open DevTools">
    Press `F12` (or `Ctrl+Shift+I` on Windows/Linux, `Cmd+Option+I` on macOS) to open browser DevTools.
  </Step>
  <Step title="Go to the Network tab">
    Click the **Network** tab at the top of DevTools. If the list is empty, refresh the page with DevTools open.
  </Step>
  <Step title="Send any message">
    In Discord, send a message in any channel or DM. This triggers an API request you can inspect.
  </Step>
  <Step title="Find the authorization header">
    In the Network tab, look for a request to `discord.com`. Click it, then click the **Headers** sub-tab. Scroll to the **Request Headers** section and find the `authorization` header. Its value is your token.
  </Step>
  <Step title="Copy the token">
    Right-click the `authorization` value and copy it. The token matches this pattern:
    ```
    [\w-]{24}\.[\w-]{6}\.[\w-]{25,110}
    ```
    It consists of three dot-separated segments. Some tokens begin with `MT`.
  </Step>
</Steps>

## Option 2: Discord desktop client

<Note>
  Extracting a token from the Discord desktop client is more involved than the browser method. Use the [token-helper script](/token-helper/auto-detect) or the browser DevTools method above instead.
</Note>

The desktop client stores tokens in LevelDB files under `%APPDATA%\discord\Local Storage\leveldb`. Encrypted tokens (prefixed with `dQw4w9WgXcQ:`) require DPAPI decryption. The `token-helper.py` script handles both steps automatically.

## Entering your token in Deleo

Once you have your token, enter it in Deleo's authentication screen:

1. Launch Deleo.
2. Choose **Enter manually** on the authentication screen.
3. Paste your token into the token field.
4. Optionally, enable **Save token** — this writes the token to `~/.deleo_cached_token` via `AuthManager.saveToken()`, so Deleo can load it automatically next time without prompting you.
5. Click **Authenticate**.

<Tip>
  If you save the token, Deleo writes it to `~/.deleo_cached_token`. On the next launch, `loadCachedToken()` reads this file automatically and skips the manual entry step.
</Tip>

## Security considerations

<Warning>
  A Discord token is equivalent to your username and password. Anyone who has your token can read your messages, send messages as you, join servers, and perform any other action your account can perform — without needing your password or two-factor code.
</Warning>

- Do not paste your token into any website or tool other than Deleo.
- Do not commit your token to version control.
- Do not share your token in screenshots or screen recordings.
- If you believe your token has been compromised, change your Discord password immediately — this invalidates the current token.
- The token-helper script only validates tokens against the official Discord API (`discord.com/api/v9`) and writes the result only to your local filesystem.
